# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

В приложении используются два интерфейса данных: для Товара и для Покупателя. Интерфейс Товара описывает всю информацию, характеризующую товар, размещённый на странице магазина.
Интерфейс Покупателя описывает всю информацию, необходимую для оформления заказа покупателем и заполняемую им после добавления Товара в Корзину.
Интерфейс описывающий каждый Товар, используется при описании двух классов, реализующих действия с товарами на странице магазина: начальное положение товаров на странице и положение товаров после их добавления в Корзину.
Интерфейс, описывающий Покупателя используется для описания класса, реализующего действия покупателя при оформлении заказа на товар.

### Интерфейс Товара

Интерфейс Товара содержит информацию об: идентификаторе товара в каталоге (или артикуле) - используется строчный тип данных, названии товара - используется строчный тип данных, изображении товара - используется строчный тип данных, категории товара - используется строчный тип данных, стоимости товара - используется числовой тип данных или null, описании товара - используется строчный тип данных. Таким образом, интерфейс, описывающий Товар, примет следующий вид:

interface IProduct {
id: string;
description: string;
image: string;
title: string;
category: string;
price: number | null;
}

### Интерфейс Покупателя

Интерфейс Покупателя содержит информацию о: вариантах оплаты товара - используется именнованный тип или null, адресе покупателя - используется строчный тип данных, электронной почте покупателя - используется строчный тип данных, контактном телефоне покупателя - используется строчный тип данных. Таким образом, интерфейс, описывающий Покупателя, примет следующий вид:

export interface IBuyer {
payment: TPayment | null;
email: string;
phone: string;
address: string;
}

## Модели данных

Модели данных отвечают за хранение данных приложения и реализованы тремя сущностями, соответствующими классам: Products, Basket и Order. Все три класса описывают бизнес-логику работы приложения, поэтому их файлы лежат в одной папке. В этой папке каждый класс оформлен в отдельном файле, название которого совпадает с названием класса. Каждый класс реализует свои уникальные методы.

### Класс Products

Описывает модель каталога товаров.

Поля класса:
`protected items: IProduct[] = []` - каталог товаров на главной странице магазина
`protected selected: IProduct | null = null` - информация о выбранной позиции из каталога

Методы:
`getItems(): IProduct[]` - получаем список товаров со страницы магазина
`setItems(items: IProduct[]): void` - сохраняем список товаров со страницы магазина
`setSelected(product: IProduct): void` - сохраняем выбранную позицию из каталога
`getSelected(): IProduct | null` - получаем выбранную позицию из каталога
`getItemById(id: string): IProduct | undefined` - получаем товар по его id

### Класс Basket

Описывает модель корзины пользователя.

Поля класса:
`protected items: IProduct[] = []` - каталог выбранных товаров с главной страницы магазина

Методы:
`addProduct(product: IProduct): void` - добавление товара в корзину
`removeProduct(id: string): void` - удаление товара из корзины
`getTotalPrice(): number` - подсчёт общей стоимости товаров в корзине
`hasProduct(id: string): boolean` - проверка доступности товара со страницы магазина для покупки
`clearBasket(): void` - очистка корзины

### Класс Order

Описывает модель оформления заказа.

Поля класса:
`protected buyer: IBuyer = {  payment:  '',  email: '',  phone:  '',  address: '' }` -
объект, заполняющийся введёнными данными покупателя:
способом оплаты, адресом электронной почты, номером телефона, физическим адресом

Методы:
`setBuyer(data: Partial<IBuyer>): void` - получение данных покупателя
`getBuyer(): IBuyer` - сохранение данных покупателя для оформления заказа
`validateOrder(): TErrors` - проверка заполненных покупателем данных
`clearOrder(): void` - очистка данных покупателя

## Слой коммуникации

Слой отвечает за взаимодействие приложения с сервером через HTTP-запросы. Коммуникация реализуется посредством класса страницы нашего конкретного магазина.

### Класс WebLarekApi

Наследуется от базового класса Api, лежит в отдельном файле в корневой папке компонентов и реализует методы работы с API интернет-магазина.

Конструктор класса:
`constructor(private api: IApi)` - принимает объект, реализующий интерфейс IApi, через который выполняются запросы к серверу.

Поля класса:
`api: IApi` - объект для выполнения HTTP-запросов.

Методы класса:
`getProducts(): Promise<IProductsResponse>` - получение каталога товаров с сервера.
`createOrder(order: IOrderRequest): Promise<IOrderResponse>` - отправляет данные сформированного покупателем заказа на сервер.

### Интерфейсы слоя коммуникации

Слой коомуникации использует в своей работе интерфейсы для обмена данными между клиентом и сервером. Интерфейсы коммуникации описаны в том же файле, что и интерфейсы, задействованные для реализации бизнес-логики.

#### IProductsResponse

Интерфейс описывает ответ сервера при запросе каталога товаров.

Поля интерфейса:
`total: number` - общее количество товаров.
`items: IProduct[]` - массив товаров.

#### IOrderRequest

Интерфейс описывает структуру данных, отправленных на сервер при оформлении заказа. Наследует поля интерфейса бизнес-логики IBuyer.

Поля интерфейса:
`items: string[]` - массив идентификаторов товаров, включённых в заказ.
`total: number` - стоимость товаров в заказе.

#### IOrderResponse

Интерфейс описывает ответ сервера после успешного оформления заказа.

Поля интерфейса:
`id: string` - идентификатор созданного заказа
`total: number` - итоговая стоимость заказа

## Слой Представления (View)
Делится на базовые и конкретные, простые и интерактивные, представления. Базовые представления: Card, Form. Простые представления: Header, Gallery, SuccessView. Интерактивные представления: GalleryCard, CardView, BasketCard, BasketView, PaymentView, ContactsView, Modal.

### Класс Header
Отвечает за отображение шапки сайта и взаимодействие пользователя с корзиной. Собственных методов не имеет.

Функциональность класса: 
1) отображение количества товаров, находящихся в корзине;
2) обработка нажатия на кнопку корзины;
3) генерация события открытия корзины.

Используемый интерфейс: 
`IHeader`

Поля интерфейса: 
`counter: number` - количество товаров, отображаемое в счётчике корзины

Поля класса:
`protected counterElement: HTMLElement;` - DOM-элемент, отображающий количество товаров в корзине.
`protected basketButton: HTMLButtonElement;` - DOM-элемент кнопки открытия корзины.

Конструктор класса:
`constructor(protected events: IEvents, container: HTMLElement)` - принимает экземпляр системы событий приложения event: IEvents и корневой DOM-элемент шапки сайта container: HTMLElement.

В конструкторе происходит поиск необходимых DOM-элементов и регистрация обработчиков пользовательских действий.

Сеттеры класса:
`set counter(value: number)` - обновляет количество товаров в корзине, используя параметр value: number.

### Класс Gallery
Отвечает за отображение каталога товаров на главной странице. Собственных полей и методов не имеет.

Функциональность класса:
1) отображение списка карточек товаров;
2) обновление содержимого каталога при получении новых данных.

Используемый интерфейс: 
`IGallery`

Поля интерфейса:
`items: HTMLElement[]` - массив HTML-элементов карточек товаров, отображаемых в каталоге.

Конструкторы класса:
`constructor(container: HTMLElement)` - принимает корневой элемент каталога товаров container: HTMLElement.

Сеттеры класса:
`set items(items: HTMLElement[])` - заменяет текущее содержимое каталога новым массивом карточек товаров.

### Базовый класс Card
Отвечает за общий функционал карточек товара. Используется как родительский класс для GalleryCard, CardView и BasketCard.

Функциональность класса:
1) поиск общих DOM-элементов карточки;
2) отображение названия товара;
3) отображение стоимости товара.

Поля класса:
`protected titleElement: HTMLElement` — DOM-элемент названия товара.
`protected priceElement: HTMLElement` — DOM-элемент стоимости товара.

Конструктор класса:
`constructor(container: HTMLElement)` — принимает корневой DOM-элемент карточки container: HTMLElement.

Сеттеры класса:
`set title(value: string)` — отображает название товара.
`set price(value: number | null)` — отображает стоимость товара. Если значение равно null, отображается строка "Бесценно".

### Класс GalleryCard
Отвечает за отображение карточки товара в каталоге. Класс наследуется от базового класса Card, из которого получает общий функционал отображения названия и стоимости товара. Собственных методов не имеет.

Функциональность класса:
1) отображение названия товара;
2) отображение изображения товара;
3) отображение стоимости товара;
4) отображение категории товара;
5) применение цветового оформления категории товара.

Используемый интерфейс:
`ICard`

Поля интерфейса:
`title: string` — название товара.
`image: string` — путь к изображению товара.
`price: number | null` — стоимость товара.
`category: string` — категория товара.

Поля класса:
`protected imageElement: HTMLImageElement` — DOM-элемент изображения товара.
`protected categoryElement: HTMLElement` — DOM-элемент категории товара.

Конструктор класса:
`container: HTMLElement, onClick: () => void` — принимает корневой DOM-элемент карточки товара container: HTMLElement и onClick: () => void - обработчик выбора товара. В конструкторе происходит поиск необходимых DOM-элементов, отвечающих только за уникальные части карточки товара.

Сеттеры класса:
`set image(value: string)` — отображает изображение товара, используя путь value: string и константу CDN_URL.
`set category(value: string)` — отображает категорию товара и применяет соответствующий CSS-класс из объекта categoryMap.

Используемые внешние зависимости:
`CDN_URL` — используется для формирования полного пути к изображению товара.
`categoryMap` — используется для сопоставления категории товара и CSS-класса оформления.

### Класс Modal
Отвечает за отображение, закрытие и управление содержимым модальных окон приложения.

Функциональность класса:
1) отображение содержимого модального окна;
2) открытие модального окна;
3) закрытие модального окна;
4) очистка содержимого модального окна после закрытия;
5) генерация события закрытия модального окна.

Используемый интерфейс:
`IModal`

Поля интерфейса:
`content: HTMLElement` — HTML-элемент содержимого модального окна.

Поля класса:
`protected closeButton: HTMLButtonElement` — DOM-элемент кнопки закрытия модального окна.
`protected contentElement: HTMLElement` — DOM-элемент области содержимого модального окна.

Конструктор класса:
`constructor(protected events: IEvents, container: HTMLElement)` — принимает экземпляр системы событий приложения events: IEvents и корневой DOM-элемент модального окна container: HTMLElement.
В конструкторе происходит поиск необходимых DOM-элементов и регистрация обработчиков пользовательских действий.

Методы класса:
`open()` — открывает модальное окно.
`close()` — закрывает модальное окно, очищает его содержимое и генерирует событие modal:close.

Сеттеры класса:
`set content(value: HTMLElement)` — заменяет текущее содержимое модального окна новым HTML-элементом.

### Класс CardView
Отвечает за отображение подробной информации о товаре в модальном окне и взаимодействие пользователя с корзиной. Класс наследуется от базового класса Card, из которого получает общий функционал отображения названия и стоимости товара.Собственных методов не имеет. 

Функциональность класса:
1) отображение названия товара;
2) отображение изображения товара;
3) отображение стоимости товара;
4) отображение категории товара;
5) отображение описания товара;
6) изменение текста кнопки добавления товара в корзину;
7) блокировка кнопки для недоступных товаров;
8) генерация события добавления или удаления товара из корзины.

Используемый интерфейс:
`ICardView`

Поля интерфейса:
`title: string` — название товара.
`image: string` — путь к изображению товара.
`price: number | null` — стоимость товара.
`category: string` — категория товара.
`description: string` — описание товара.
`buttonText: string` — текст кнопки взаимодействия с корзиной.

## Слой Презентера (Presenter)
Слой реализован в файле main.ts и является единственной точкой координации приложения в архитектуре MVP. Presenter связывает между собой модели, представления, API и систему событий. Не хранит данные приложения и не содержит бизнес-логику самостоятельно, а управляет взаимодействием между слоями приложения.

Функциональность Presenter:
1) создаёт экземпляры моделей, представлений и API-классов;
2) загружает товары с сервера;
3) создаёт карточки товаров для каталога;
4) обрабатывает пользовательские события;
5) передаёт действия пользователя в модели;
6) реагирует на события моделей и обновляет представления;
7) управляет открытием и содержимым модальных окон;
8) выполняет отправку заказа на сервер;
9) очищает корзину и данные заказа после успешного оформления.

### Основные сущности Presenter
Система событий:
`const events = new EventEmitter()` — единая система событий приложения, через которую представления сообщают о действиях пользователя.

API:
`const api = new Api(API_URL)` — базовый API-класс для выполнения запросов.
`const webApi = new WebLarekApi(api)` — API-класс приложения, который получает товары и отправляет заказы.

Модели:
`const productsModel = new Products(events)` — модель списка товаров.
`const basketModel = new Basket(events)` — модель корзины.
`const orderModel = new Order(events)` — модель данных заказа.

Представления:
`header` — представление шапки сайта.
`gallery` — представление каталога товаров.
`modal` — универсальное модальное окно.

Вспомогательные функции Presenter:
`cloneTemplate(template: HTMLTemplateElement): HTMLElement` — клонирует HTML-шаблон и возвращает его корневой DOM-элемент.
`getOrderState()` — возвращает текущее состояние заказа: данные покупателя и ошибки валидации.
`renderPaymentForm(): void` — перерисовывает форму оплаты и адреса после изменения полей.
`renderContactsForm(): void` — перерисовывает форму контактных данных после изменения email или телефона.
`createCatalogCard(item: IProduct): HTMLElement` — создаёт карточку товара для каталога и регистрирует обработчик выбора товара.
`createBasketCard(item: IProduct, index: number): HTMLElement` — создаёт элемент товара в корзине.
`renderBasket(): void` — перерисовывает корзину на основе актуального состояния модели Basket.

Основные события Presenter:
`products:changed` — обновляет витрину товаров после изменения модели Products.
`product:select` - сообщает о выборе товара пользователем и передаёт выбранный товар в модель Products.
`product:selected` — открывает подробную карточку товара в модальном окне после сохранения выбранного товара в модели Products.
`order:changed` — обновляет формы оформления заказа после изменения модели Order.
`basket:open` — открывает корзину.
`basket:toggle` — добавляет товар в корзину или удаляет его из корзины.
`basket:remove` — удаляет товар из корзины.
`basket:changed` — обновляет счётчик корзины и содержимое корзины, если она открыта.
`order:start` — открывает форму выбора оплаты и ввода адреса.
`payment:change` — сохраняет выбранный способ оплаты.
`address:change` — сохраняет адрес доставки.
`order:submit` — проверяет данные оплаты и адреса, затем открывает форму контактов.
`contacts.email:change` — сохраняет email покупателя.
`contacts.phone:change` — сохраняет телефон покупателя.
`contacts:submit` — проверяет весь заказ и отправляет его на сервер.
`order:success` — отображает окно успешного оформления заказа и очищает состояние корзины и заказа.
`success:close` — закрывает модальное окно успешного оформления заказа.

## Пользовательский сценарий работы приложения
1) При загрузке приложения Presenter получает товары с сервера через webApi.getProducts().
2) Полученные товары сохраняются в productsModel.
3) Модель Products генерирует событие products:changed.
4) Presenter реагирует на событие products:changed, создаёт карточки GalleryCard и передаёт их в Gallery.
5) При выборе товара данные сохраняются в модели Products.
6) Модель Products генерирует событие product:selected.
7) Presenter открывает CardView в модальном окне.
8) При добавлении или удалении товара Presenter передаёт действие в basketModel.
9) Модель Basket генерирует событие basket:changed.
10) Presenter обновляет счётчик в Header и содержимое корзины, если она открыта.
11) При открытии корзины Presenter создаёт BasketView.
12) При оформлении заказа Presenter открывает PaymentView.
13) После выбора оплаты и ввода адреса Presenter открывает ContactsView.
14) После ввода контактов Presenter отправляет заказ через webApi.createOrder().
15) После успешного ответа сервера Presenter открывает SuccessView.
16) Корзина и данные заказа очищаются.
